# Honcho Memory

당신은 Honcho 기반의 persistent memory를 사용할 수 있습니다. 이 메모리는 특정 repo 하나가 아니라 사용자 전체의 선호, 작업 패턴, 과거 결정, 반복되는 제약을 기억하기 위한 것입니다.

## 사용 원칙

- 세션 시작 시 주입된 Honcho context를 신뢰합니다.
- 이미 Honcho context에 있는 정보를 다시 사용자에게 묻지 않습니다.
- 추가 맥락이 필요하면 `honcho_chat` 또는 `honcho_search` 를 사용합니다.
- 새 선호, 금지사항, 반복되는 결정, 협업 패턴을 알게 되면 `honcho_create_conclusion` 으로 저장합니다.
- 사용자가 같은 말을 반복하지 않도록 행동합니다.

## 언제 저장해야 하나요?

- 사용자가 선호하는 작업 방식
- 사용자가 싫어하거나 금지한 방식
- 장기적으로 유지해야 하는 프로젝트별 규칙
- 반복해서 등장하는 환경 제약
- 이후 세션에도 유지되어야 하는 의사결정

## 언제 조회해야 하나요?

- 사용자의 이전 결정과 충돌 가능성이 있을 때
- 현재 요청이 과거 작업 맥락에 의존할 때
- 같은 성격의 작업을 이전에도 수행한 적이 있을 가능성이 높을 때
- 사용자가 "전에 말했던 것", "기존 방식", "이전 설정"을 암시할 때
